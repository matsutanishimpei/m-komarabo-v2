# セキュリティポリシー (Security Policy)

「松谷の試作室」プロジェクトへ関心をお寄せいただき、ありがとうございます。

## セキュリティに関する考え方

現在はプロトタイプ段階（初期開発フェーズ）ですが、ログイン認証（OAuth 2.0 + JWT）や D1 データベースの操作については、可能な限り標準的なセキュリティ対策を施しています。

具体的には以下のような対策を含んでいます：
- JWT の署名検証と HttpOnly Cookie によるセッション管理
- ユーザー識別子（sub値）に基づく厳密なデータ所有権チェック
- SQL インジェクションを防ぐためのクエリのパラメーター化（D1 APIの標準仕様の遵守）
- 秘密情報（Secret）の環境変数管理

## 脆弱性の報告方法

もし本プロジェクトにセキュリティ上の脆弱性（不具合）を発見された場合は、**パブリックな ISSUE を作成せず**、以下の連絡先へ直接ご連絡ください。

- 報告先: **GitHub の公開メールアドレス** または **作者（松谷）の連絡先** へ直接メッセージをお願いします。

ご報告いただいた内容については、速やかに確認を行い、必要に応じて修正パッチを提供いたします。
